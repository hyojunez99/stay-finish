# 주차장 관리 웹 사이트

## 1. 프로젝트 소개

- **설명:**  
입주민 · 상가 사용자 · 관리자 역할에 따라 주차 현황을 관리하는 웹 애플리케이션으로, 방문 차량 등록부터 할인권 발급·정산까지의 흐름을 React 기반으로 구현한 팀 프로젝트입니다.

- **진행 기간:** 2025.01.02 ~ 2025.01.13 (12일)  
- **개발 인원:** 6인 팀 프로젝트  
- **배포 링크:** [Stay-Finish](https://hyojunez99.github.io/stay-finish/)  
- **GitHub:** [Repository Link](https://github.com/hyojunez99/stay-finish.git)  
- **PDF:** [stay.pdf](stay.pdf)

---

## 2. 사용 기술 스택 (Tech Stack)

- **Frontend:** JavaScript (ES6+), React.js, SCSS  
- **Backend:** Supabase  
- **Tools:** Figma, Photoshop, Illustrator, Git/GitHub, GSAP  

---

## 3. 기획 및 디자인 (Planning & Design)

- **컨셉:**  
입주민, 상가 사용자, 관리자 역할에 따라 주차 현황과 업무 흐름을 직관적으로 관리할 수 있는 웹 애플리케이션  

- **Tool:** Figma  
- **주요 활동:**
  - 전체 화면 및 공통 UI 설계 및 구현  
  - 역할별 화면 분기 및 기능 설계  
  - 팀 협업을 위한 Git 브랜치 관리 및 소스 코드 병합(Merge) 담당

## 4. 디렉토리 구조

```text
src
├── api             # Supabase 기반으로 관리자, 사용자, 주차, 정산 등 주요 기능의 데이터 처리를 담당하는 API 모음
│   
├── assets          # 프로젝트 전반에서 공통으로 사용하는 리소스를 관리
│     │   
│     ├── images    # 페이지 각 섹션에 사용되는 이미지 파일을 관리
│     │   
│     ├── scss      # 공통 스타일 정의를 위한 색상 변수, 폰트 스타일, 반응형 및 레이아웃 mixin, 전역 스타일을 관리
│     │   
│     └── svg       # 페이지에 사용되는 SVG 아이콘 및 이미지 파일을 관리
│     
├── components      # 공통 UI 및 서비스 핵심 기능 컴포넌트를 관리
│     │    
│     ├── common    # 프로젝트 전반에서 공통으로 사용되는 UI 및 스크립트 컴포넌트를 관리
│     │   
│     └── mode      # 차량 등록, 할인권 발급 및 정산 등 주차장 관리 핵심 기능 컴포넌트를 관리
│     
├── contexts        # 사용자, 관리자, 주차 현황 등 도메인별 상태 관리
│
├── layout          # 공통 페이지 레이아웃 관리
│
└── pages           # 실제 보여지는 페이지 컴포넌트 관리

```

## 7. 트러블 슈팅

- **문제 1:** 페이지 새로고침 시 사용자 프로필 정보가 초기화되는 문제
  - **해결:** useEffect에서 profile.id가 준비된 시점에 헤더 영역의 사용자 정보를 다시 요청하도록 로직을 수정하여 상태를 복원

- **문제 2:** 협업 과정에서 필요한 API 스펙을 사전에 공유하지 못해 개발 흐름이 지연
  - **해결:** API 구조를 기준으로 화면 로직을 재정비하고, 이후 API 기준으로 기능을 구현하는 방식으로 협업 방식을 개선

- **문제 3:** 구현 도중 일부 페이지를 추가해야 하는 상황 발생
  - **해결:** 전체 구조 변경 대신 공통 레이아웃과 라우팅 구조를 정리하여 유연하게 페이지 추가 가능하도록 개선

## 8. 인사이트 (Insights)

- 필요한 API를 사전에 충분히 공유하지 못하면 개발 과정에서 한계를 느낄 수 있음
- 화면 흐름을 기준으로 필요한 API를 먼저 정리하고, 관련 내용을 미리 소통하는 것이 중요
- 팀 프로젝트 협업 시 Git 브랜치 관리와 코드 병합 전략의 중요성 체감
